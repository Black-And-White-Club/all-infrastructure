### ⚠️ cert-manager: v1.16.2 → v1.20.3 — Potentially breaking changes across four minor versions

cert-manager was updated across four minor versions. Individually low-risk, but operators should be aware:

**v1.17.0:**
- Potentially BREAKING: CA and SelfSigned issuers now use SHA-512 when signing RSA keys of 4096 bits or above (was SHA-256). Affects certificates issued with large RSA keys.
- Potentially BREAKING: Log messages converted to structured JSON format. May break log-parsing pipelines.
- Feature gates `NameConstraints` and `UseDomainQualifiedFinalizer` promoted to beta (now on by default).
- `ValidateCAA` feature gate deprecated (removed in v1.18.0).

**v1.18.0:**
- Potentially BREAKING: Default `revisionHistoryLimit` for CertificateRequest revisions changed to 1 (was unlimited). May affect workflows that rely on retaining multiple CertificateRequest revisions.
- Fix for Cloudflare DNS01 challenges after a breaking change to the Cloudflare API.

**v1.19.0:**
- Major upgrade to Akamai SDK (not fully tested in upstream).

**v1.20.3 (target) — SECURITY FIX (HIGH): GHSA-8rvj-mm4h-c258**
> "The default `cert-manager-edit` aggregate ClusterRole granted namespace users permission
> to create ACME `Challenge` and `Order` resources directly. A user who could create a
> `Challenge` referencing a `ClusterIssuer` could supply attacker-controlled solver configuration
> while cert-manager loaded credentials from the `ClusterIssuer`'s namespace, bypassing Issuer
> solver selectors. With the acme-dns provider specifically, this could disclose DNS credentials
> to an attacker-controlled endpoint."
>
> **All users should upgrade.** This update is included in this batch.
