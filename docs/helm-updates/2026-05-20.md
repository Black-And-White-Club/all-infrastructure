## Changelogs

### cert-manager: v1.16.2 → v1.20.2

**GitHub releases:** https://github.com/cert-manager/cert-manager/releases

This update spans **four minor versions** (v1.17, v1.18, v1.19, v1.20) each of which contains
notable breaking or behaviorally significant changes. Review before merging.

#### v1.17.0 — Breaking Changes
- **RSA hash algorithm change**: CA and SelfSigned issuers now use SHA-512 for RSA keys ≥4096 bits
  and SHA-384 for RSA keys ≥3072 bits. Systems using large RSA keys must verify support.
- **Structured logging**: Log message format changed from unstructured to structured. Any log
  parsing/alerting rules on cert-manager logs may break.
- **ACME ClusterIssuer namespace bug fixed**: A v1.16.0 regression where ACME ClusterIssuers
  searched the wrong namespace has been corrected (behavior change from buggy state).
- New Helm values: `webhook.extraEnv`, `cainjector.extraEnv`, `startupapicheck.extraEnv`

#### v1.18.0 — Breaking Changes
- **`RotationPolicy` default changed**: `Certificate.Spec.PrivateKey.RotationPolicy` now defaults
  to `Always` (was `Never`). Existing certificates will begin rotating private keys automatically.
  If you rely on key stability (e.g., for MTLS pinning), set `rotationPolicy: Never` explicitly.
- **`RevisionHistoryLimit` default changed to 1**: Fewer CertificateRequest revisions retained.
- New security option: `global.rbac.disableHTTPChallengesRole` (removes HTTP-01 ACME pod-create
  privileges when not needed).

#### v1.19.0 — Breaking Changes
- **ACME metric label removed**: The `path` label was removed from core ACME client metrics.
  Update Prometheus dashboards and alerting rules that query `path`-labeled ACME metrics.
- Known issue: Unexpected certificate renewal after upgrade (fixed in v1.19.1).
- The `global.rbac.disableHTTPChallengesRole` option added in v1.18.0 was reverted in v1.19.0.

#### v1.20.0 — Breaking Changes
- **Container UID/GID changed**: Default container user is now `65532` (was `1000`) and default
  group is `65532`. This may cause volume permission issues if volumes are owned by UID 1000.
- **`DefaultPrivateKeyRotationPolicyAlways` feature gate promoted to GA**: This gate can no longer
  be disabled; key rotation policy `Always` is now permanently the default.
- Azure Private DNS support added for DNS-01 issuers.
- `OtherNames` feature promoted to Beta (enabled by default).

#### v1.20.2 (patch — no breaking changes)
- Fixes invalid YAML generated when both `webhook.config` and `webhook.volumes` are defined.
- Go bumped to 1.26.2 for dependency security patches.
