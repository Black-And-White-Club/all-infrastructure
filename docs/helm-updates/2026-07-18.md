### ⚠️ argo-cd 10.0.0 Breaking Change: NetworkPolicy default enabled

Chart 10.0.0 changed `global.networkPolicy.create` from `false` to `true`. This enables Kubernetes
NetworkPolicies for Argo CD components by default. If the cluster uses a CNI that enforces
NetworkPolicies (Cilium, Calico, etc.), this could restrict traffic between Argo CD components and
external services. If you use a service mesh or custom NetworkPolicy type (e.g., CiliumNetworkPolicy),
set `global.networkPolicy.create: false` in your values file.

Release notes: https://github.com/argoproj/argo-helm/releases/tag/argo-cd-10.0.0
