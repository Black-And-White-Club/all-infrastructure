### cert-manager: Breaking Changes Detail (v1.18+)

Upgrading from v1.16.2 to v1.20.2 spans three minor versions. The following **breaking changes** were introduced in v1.18.0 and must be reviewed before applying:

1. **`Certificate.Spec.PrivateKey.RotationPolicy` now defaults to `Always`** (was `Never`). This means cert-manager will now automatically rotate private keys on every certificate renewal by default. If you have existing certificates with explicit `rotationPolicy: Never` this is fine, but any `Certificate` resource without an explicit `rotationPolicy` will now rotate its key on next renewal. Review all `Certificate` resources.

2. **`Certificate.Spec.RevisionHistoryLimit` now defaults to `1`** (was `nil`/unlimited). Old `CertificateRequest` objects older than the limit will be garbage-collected. Potentially breaking if anything depends on the CertificateRequest history.

3. **Known issue in v1.18**: ACME HTTP01 challenge paths are rejected by the ingress-nginx validating webhook (see cert-manager issue tracker). May affect certificate issuance if using ACME HTTP01 with ingress-nginx.

4. **Known issue in v1.19.0**: Unexpected certificate renewal immediately after upgrade (fixed in v1.19.1 — not a concern since we're going to v1.20.2).
