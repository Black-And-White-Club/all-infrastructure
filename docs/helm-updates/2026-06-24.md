**⛔ DO NOT APPLY THIS UPDATE WITHOUT A MIGRATION PLAN.**

The bitnami/postgresql chart version `16.2.4` runs **PostgreSQL 17.2.0** (note: chart major version ≠ PG major version in bitnami's scheme). The latest chart `18.7.8` runs **PostgreSQL 18.4.0**. A PostgreSQL major version upgrade causes `initdb` to run on startup, which **WIPES THE ENTIRE FROLF DATABASE**. This already happened once before.

**Safe in-series update available (but not applied per policy):** chart `16.7.27` → PG `17.6.0` would stay on the same PG major version and would be safe. Requires explicit human decision before applying.

Required before upgrading to PG 18:
- Schedule a maintenance window
- Back up the frolf database
- Plan a `pg_upgrade` or dump/restore migration
- Test against a replica first
