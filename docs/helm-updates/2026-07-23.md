### alloy: 1.8.1 → 1.11.0

**Release context:** Chart version 1.11.0 ships Grafana Alloy v1.18.0 (released 2026-07-17).

**⚠️ Breaking changes in Alloy v1.18.0:**

1. **HTTP server timeouts:** `otelcol.receiver.otlp` (and faro, jaeger, zipkin, influxdb, splunkhec, datadog) and `otelcol.extension.jaeger_remote_sampling` now default `idle_timeout = "1m"`, `read_header_timeout = "1m"`, and `write_timeout = "30s"`. Previously defaulted to `"0s"`. Set explicitly to `"0s"` in config to restore prior behavior.

2. **Kafka receiver/exporter:** `otelcol.receiver.kafka` and `otelcol.exporter.kafka` — `resolve_canonical_bootstrap_servers_only` is now a no-op and deprecated (migrated from sarama to franz-go). Still parses but no longer resolves bootstrap addresses.

3. **splunkhec exporter:** `otelcol.exporter.splunkhec` — `batcher` block removed (upstream deletion in otel-contrib v0.151). Configs using it will fail to load. Migrate: move fields under `sending_queue.batch`.

These breaking changes only affect Alloy configurations that use the specific components listed. Review your Alloy config in `charts/alloy/values.yaml`.

**GitHub releases:** https://github.com/grafana/alloy/releases/tag/helm-alloy-1.11.0
