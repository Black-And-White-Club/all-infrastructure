### Applied — Breaking Changes Requiring Review Before Merging

#### argo-cd: 9.5.14 → 10.1.3 (⚠️ MAJOR chart version)

The argo-cd Helm chart 10.0.0 introduced the following breaking changes that **must be reviewed against this installation before merging**:

1. **`server.additionalApplications` and `server.additionalProjects` removed** — These values were moved to the separate `argocd-apps` chart. Check `charts/argo-cd/values.yaml` for any definitions using these keys.
2. **Ingress catch-all rule removed** — The catch-all ingress rule was removed for security reasons. If a catch-all rule is needed, it must be added via `server.ingress.extraRules`.
3. **CRDs moved** — CRDs were moved from `<chart>/crds/` to `<chart>/templates/` for proper upgrade support. ArgoCD uses `ServerSideApply=true` already, which should handle this, but verify.
4. **redis-ha haproxy deployment** — If `redis-ha` is enabled, the argocd-redis-ha-haproxy deployment requires replacement (immutable selector label change). Check current values: `redis-ha.enabled`.
5. **ArgoCD appVersion** — Updated to v3.4.5 (from the v2.x series likely pinned in the old chart — check `spec.source.helm.parameters` for any appVersion pins).

#### cert-manager: v1.16.2 → v1.21.0 (5 minor versions)

cert-manager v1.21.0 breaking changes that **must be reviewed against `charts/cert-manager/values.yaml`**:

1. **RBAC removed** — The default Role/RoleBinding granting the cert-manager controller `serviceaccounts/token: create` was removed (added in v1.16, now removed). If using `serviceAccountRef.name` pointing at the controller SA, create your own RBAC or migrate to IRSA ambient patterns.
2. **Metrics values removed** — `prometheus.servicemonitor.targetPort`, `prometheus.servicemonitor.path`, and `prometheus.podmonitor.path` have been removed. The controller Service metrics port was renamed from `tcp-prometheus-servicemonitor` to `http-metrics`. Update your values overrides if any of these are set — the chart will schema-validate and reject the upgrade otherwise.
3. **Intermediate releases (v1.17–v1.20)** — There may be additional breaking changes from the skipped intermediate versions. Review the cert-manager release notes for v1.17, v1.18, v1.19, v1.20 before merging. Start at: https://cert-manager.io/docs/releases/
