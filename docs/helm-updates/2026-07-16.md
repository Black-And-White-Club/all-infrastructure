### argo-cd: 9.5.14 → 10.1.3

**Key breaking changes in v10.0.0:**
- `global.networkPolicy.create` changed from `false` to `true` by default. If using a service mesh or Cilium, set `global.networkPolicy.create: false` in values to avoid traffic disruption.

### cert-manager: v1.16.2 → v1.21.0

**GitHub:** https://github.com/cert-manager/cert-manager/releases

**Key breaking changes (cumulative v1.17 through v1.21):**

- **v1.17:** Unstructured log messages replaced with structured logs. Setups matching on specific log strings may need updating.
- **v1.18:** `Certificate.Spec.PrivateKey.RotationPolicy` default changed to `Always` (previously `Never`). `Certificate.Spec.RevisionHistoryLimit` default changed to `1`. ACME HTTP01 `PathType` changed to `Exact`. Review existing Certificate resources for unexpected re-issuance.
- **v1.19:** High-cardinality `path` label removed from ACME metrics (`certmanager_acme_client_request_count`, `certmanager_acme_client_request_duration_seconds`), replaced with `action`. Update Grafana dashboards and alerting rules accordingly.
- **v1.21:** Helm chart no longer ships a default Role/RoleBinding for the controller ServiceAccount to create tokens. If using `serviceAccountRef.name` pointing at the controller SA, create your own Role/RoleBinding or migrate. Prometheus values `prometheus.servicemonitor.targetPort`, `prometheus.servicemonitor.path`, `prometheus.podmonitor.path` removed; metrics service port renamed from `tcp-prometheus-servicemonitor` to `http-metrics`.
