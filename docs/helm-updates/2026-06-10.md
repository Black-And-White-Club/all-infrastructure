### ⚠️ cert-manager Breaking Changes (v1.16.2 → v1.20.2)

This update spans 4 minor versions and includes breaking changes introduced between v1.17.0 and v1.20.0. **Review before merging:**

**v1.17.0 breaking changes:**
- RSA certificate signing algorithm changed: SHA-384 for 3072-bit keys, SHA-512 for 4096-bit+ keys (previously always SHA-256). If using a large RSA CA, verify downstream systems support the new hash algorithms.
- Log messages converted from unstructured to structured format. Log-scraping pipelines may need updating.
- `ValidateCAA` feature gate deprecated (removed in v1.18).

**v1.20.0 breaking changes:**
- **Default container UID/GID changed from 1000:0 → 65532:65532.** Review PodSecurityPolicies, SecurityContextConstraints, volume permissions, and network policies referencing the old UID/GID.
- `DefaultPrivateKeyRotationPolicyAlways` feature gate promoted to GA — it can no longer be disabled.
- API defaults for issuer reference kind/group reverted (fixes regression from v1.19.x that caused unexpected certificate renewal on upgrade).
