## cert-manager Breaking Changes Detail (1.16.2 → 1.20.2)

The jump from v1.16.2 to v1.20.2 crosses cert-manager v1.18.0, which introduced several breaking changes:

1. **`Certificate.Spec.PrivateKey.RotationPolicy` default changed** from `Never` → `Always`. If certificates do not explicitly set `rotationPolicy`, they will now rotate the private key on each renewal. Existing Certificates should be audited and `rotationPolicy: Never` explicitly set if the old behavior is required.

2. **ACME HTTP01 `PathType` changed to `Exact`**. This is incompatible with `ingress-nginx >= 1.8.0` when `strict-validate-path-type` is enabled (it disallows `.` in path values, breaking `.well-known/acme-challenge/` paths). Verify your ingress-nginx configuration before applying. The current ingress-nginx version (4.15.1) should be checked for this setting.

3. **`Certificate.Spec.RevisionHistoryLimit` default now `1`** (potentially breaking). Previous behavior kept unlimited history.

4. **OperatorHub packages discontinued** as of v1.16.5.

**Recommended action**: Review all `Certificate` resources for explicit `rotationPolicy` settings and confirm `ingress-nginx` is not using `strict-validate-path-type` before merging this cert-manager update.
