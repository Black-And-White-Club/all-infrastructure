**App release notes** ([view on GitHub](https://github.com/argoproj-labs/argocd-image-updater/releases/tag/v1.2.0)):

The upstream app CHANGELOG notes the following changes introduced in the v1.2.0 era:
- `argocd-image-updater` now uses the **K8s API by default** to retrieve applications, instead of the Argo CD API. To retain previous behaviour, set `applications_api: argocd` in the `argocd-image-updater-config` ConfigMap before updating.
- It is now recommended to install in the same namespace as Argo CD (`argocd` by default).
- `argocd-image-updater-sa` ServiceAccount permissions updated to allow read access to `Secret` and `argoproj.io/Application` resources.
- Bug fix: installs missing `git` binary.
- Bug fix: runs `git add` for created files before pushing back.
- New: supports managing Application CRD using K8S client.
- New: allows reuse of Argo CD repo credentials.
- New: Git write-back of parameters.

**⚠️ Note for this deployment:** The installation is in the `argocd` namespace (confirmed in the chart values), which aligns with the recommended configuration. No additional action should be needed, but verify the `argocd-image-updater-config` ConfigMap does not have `applications_api: argocd` set if you want to use the new default K8s API mode.
