### cert-manager: v1.16.2 → v1.20.3

**Release notes** ([view on GitHub](https://github.com/cert-manager/cert-manager/releases/tag/v1.20.3)):

This patch release fixes a security issue ([`GHSA-8rvj-mm4h-c258`](https://github.com/cert-manager/cert-manager/security/advisories/GHSA-8rvj-mm4h-c258), **HIGH**) where the default `cert-manager-edit` aggregate ClusterRole granted namespace users permission to create ACME Challenge and Order resources directly.

**Potentially breaking change:** The `cert-manager-edit` aggregate ClusterRole no longer grants `create` for `challenges.acme.cert-manager.io` or `create`, `patch`, `update` for `orders.acme.cert-manager.io`. If you have tooling that creates Challenge or Order resources directly (outside the normal Certificate→CertificateRequest→Order→Challenge flow), you will need to grant those permissions explicitly.

Also removes issuer owner reference from Challenges which was blocking garbage collection, and updates Go to fix reported CVEs.

**All users should upgrade** (security fix).

The range v1.17.0 through v1.20.2 include additional feature releases. See cert-manager [v1.17.0 release notes](https://github.com/cert-manager/cert-manager/releases/tag/v1.17.0) for the full set of changes since v1.16.2.
