| cert-manager | argocd/platform/cert-manager.yaml | v1.16.2 | v1.20.2 | SKIPPED — 4 minor version jump with documented breaking changes. See details below. |

**cert-manager v1.16.2 → v1.20.2 breaking changes summary:**

- **v1.17**: RSA certificates with keys ≥ 3072 bits now default to SHA-384/SHA-512 hash algorithms. Structured logging replaces unstructured logs (breaks log scrapers matching on specific strings).
- **v1.18**: `Certificate.Spec.PrivateKey.RotationPolicy` default changed from `Never` to **`Always`** — certificates may be **automatically rotated on upgrade**. `RevisionHistoryLimit` default changed to 1.
- **v1.19**: High-cardinality `path` label removed from `certmanager_acme_client_request_count` and `certmanager_acme_client_request_duration_seconds` metrics; replaced with bounded `action` label — **dashboards and alerts will break**.
- **v1.20**: Review release notes before applying.

**Recommendation:** Upgrade one minor version at a time (v1.16 → v1.17 → v1.18 → v1.19 → v1.20), reviewing each set of breaking changes and updating Grafana dashboards/alerts for the metric rename. The RotationPolicy change in v1.18 is the highest-risk item — certificates in the cluster may be re-issued on upgrade.
