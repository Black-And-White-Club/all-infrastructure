### cert-manager: v1.16.2 → v1.20.2

**Release notes** ([view on GitHub](https://github.com/cert-manager/cert-manager/releases/tag/v1.20.2)):

⚠️ **This is a 4-minor-version jump. Review each intermediate release's migration guide before applying.**

**v1.17 breaking changes:**
- Log messages migrated to structured logging — may break setups matching on specific log strings
- `ValidateCAA` feature gate deprecated (removed in v1.18); enabling it now prints a warning

**v1.18 breaking changes:**
- `Certificate.Spec.PrivateKey.RotationPolicy` default changed to `Always` — certificates will now automatically rotate private keys on renewal by default. Review any certificates where you rely on the old default (`Never`)
- `Certificate.Spec.RevisionHistoryLimit` default changed to `1` — old CertificateRequest objects will be pruned more aggressively
- ACME HTTP01 Ingress `PathType` changed to `Exact` for security

**v1.19 breaking changes:**
- High-cardinality `path` label removed from `certmanager_acme_client_request_count` and `certmanager_acme_client_request_duration_seconds` metrics; replaced with bounded `action` label — **update any Grafana dashboards or Prometheus alerts that query these metrics**

**v1.20 new features:**
- Gateway API ListenerSet support for creating Certificates via annotation

Full upgrade guide: https://cert-manager.io/docs/installation/upgrade/
