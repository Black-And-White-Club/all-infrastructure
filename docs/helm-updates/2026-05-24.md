frolf-bot already lost its entire database once due to this exact failure mode (PR #10 caused the PG 16→17 transition). The `PINNED` comment in `frolf-postgres.yaml` explicitly documents this risk.

**What is required before upgrading:**
1. Schedule a maintenance window
2. Take a full, verified backup of the frolf-bot database
3. Follow the [PostgreSQL major-version upgrade guide](https://www.postgresql.org/docs/current/upgrading.html) (pg_upgrade or dump/restore)
4. Update the chart version and test recovery before applying to production

**DO NOT merge this file with a version bump without completing the above steps.**
