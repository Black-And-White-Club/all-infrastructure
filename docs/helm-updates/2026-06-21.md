### ⚠️ cert-manager — Breaking Changes Across 4 Minor Versions

Upgrading cert-manager from v1.16.2 to v1.20.2 spans four minor releases. Known breaking changes:

**v1.17.0** (2025-02-03):
- RSA signature algorithm changed: 3072-bit keys now use SHA-384, 4096-bit keys now use SHA-512 (was always SHA-256). Downstream systems must support these hash algorithms.
- Structured logging: log messages are now structured; any log parsing on specific string formats will break.
- ACME ClusterIssuer namespace bug fix (may affect existing ClusterIssuers using credential secrets).

**v1.18.0** (2024-06-10):
- `Certificate.Spec.PrivateKey.RotationPolicy` default changed from `Never` to `Always`. Existing certificates will have their private keys rotated on next renewal unless this is explicitly set back to `Never`.
- `CertificateRequest` revision history limit default set to 1.

**v1.19.0** (2024-10-07):
- `path` label removed from core ACME client metrics — monitoring dashboards and alerting rules using this label will need updating.

**v1.20.2** (2026-04-11):
- Bug fix only: invalid YAML when both `webhook.config` and `webhook.volumes` are defined. No additional breaking changes.

**Recommendation:** Review the cert-manager upgrade docs at https://cert-manager.io/docs/releases/ before merging, especially the private key rotation policy change.
