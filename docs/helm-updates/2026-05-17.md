## Changelogs

### cert-manager: v1.16.2 → v1.20.2

**⚠️ Important upgrade note:** cert-manager's official docs recommend upgrading **one minor version at a time** in production. This commit jumps 4 minor versions (1.16 → 1.17 → 1.18 → 1.19 → 1.20). Review the upgrade guide for each intermediate version before merging. The CRD bundle (`crds.enabled=true`) is already set in our values, so CRD migrations should apply automatically.

**Notable changes across intermediate versions:**

- **v1.17.0** — RSA certificates: 3072-bit keys now use SHA-384; 4096-bit keys use SHA-512 (was SHA-256). Structured logging replaces old log format (may affect log-scraping pipelines). `ValidateCAA` feature gate deprecated. IPv6 ACME HTTP-01 solver support added.
- **v1.18.x / v1.19.x** — Security and stability patches. See GitHub releases for details.
- **v1.20.0** — New feature: specify PKCS#12 and JKS keystore passwords as literal values in Certificate resources (no separate Secret required). `NameConstraints` and `UseDomainQualifiedFinalizer` features promoted to beta (enabled by default).
- **v1.20.2** — Helm chart bug fix: invalid YAML when `webhook.config` and `webhook.volumes` both defined. Go 1.26.2 + security dependency updates.

**GitHub releases:** https://github.com/cert-manager/cert-manager/releases/tag/v1.20.2

**Upgrade guide:** https://cert-manager.io/docs/installation/upgrading/
