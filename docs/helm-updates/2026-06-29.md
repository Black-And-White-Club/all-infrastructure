### cert-manager: v1.16.2 → v1.20.3

**Release notes** ([view on GitHub](https://github.com/cert-manager/cert-manager/releases/tag/v1.20.3)):

> **Security fix (HIGH — GHSA-8rvj-mm4h-c258):** The `cert-manager-edit` aggregate ClusterRole previously granted namespace users permission to create ACME `Challenge` and `Order` resources directly. A user who could create a `Challenge` referencing a `ClusterIssuer` could supply attacker-controlled solver configuration while cert-manager loaded credentials from the `ClusterIssuer`'s namespace, bypassing Issuer solver selectors. With the acme-dns provider specifically, this could disclose DNS credentials to an attacker-controlled endpoint. All users should upgrade.
>
> **Potentially breaking change:** The `cert-manager-edit` aggregate ClusterRole no longer grants `create` for `challenges.acme.cert-manager.io` or `create`, `patch`, `update` for `orders.acme.cert-manager.io`. If you have tooling that creates Challenge or Order resources directly, you will need to grant those permissions explicitly.
>
> Also fixes: issuer owner reference from Challenges blocking garbage collection; Go CVEs.
