## Methodology Note

Helm was not available in this environment and the Helm chart repository HTTP endpoints were unreachable due to egress policy restrictions (`charts.bitnami.com`, `*.github.io`, etc., returned 403). Versions were determined by:

1. **GitHub release tags** for all charts where a tag pattern was documented (sealed-secrets, ingress-nginx, argo-*, cert-manager, tailscale, loki, kube-state-metrics, alloy, prometheus-node-exporter, tempo-distributed, mimir-distributed, grafana, nats)
2. **GitHub raw file contents** (`raw.githubusercontent.com`) to verify `appVersion` (PostgreSQL version) for bitnami/postgresql chart versions
3. **Docker Hub OCI registry** (`hub.docker.com`) for bitnami/postgresql chart 18.x, which is only distributed via OCI

For the bitnami/postgresql frolf-postgres safety check: the GitHub source (bitnami/charts, last tagged `17.1.0` = appVersion `17.6.0` = PostgreSQL 17) and the OCI registry (latest `18.7.8`, appVersion unknown) diverge. Since the PostgreSQL major version for chart 18.x cannot be confirmed via available API access, frolf-postgres is conservatively skipped.
